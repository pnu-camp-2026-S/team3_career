# 3번 1페이지 요약형 디자인 기준

## 1. 템플릿 목적

1페이지 요약형은 사용자의 핵심 경험과 역량을 한 장에서 빠르게 읽을 수 있도록 정리하는 PPT 형식이다.

긴 포트폴리오를 보기 전, 지원자의 기본 정보와 핵심 강점을 한눈에 파악하도록 돕는다. 발표용 상세 설명보다 빠른 스캔과 첫인상 전달을 우선한다.

## 2. 전체 구조

1페이지 요약형은 16:9 PPT 한 장으로 구성한다.

| 영역 | 역할 |
|---|---|
| 상단 | 포트폴리오 라벨, 제목, 페이지 번호 |
| 좌측 상단 | 프로필 사진, 이름, 희망 직무, 연락처 |
| 우측 상단 | 핵심 경험 2~3개 |
| 좌측 하단 | 보유 역량 키워드 |
| 우측 하단 | 기술·도구 또는 자격·교육 |
| 하단 | 지원자를 설명하는 한 줄 소개 |

## 3. 디자인 방향

- 출력 형식은 16:9 가로형 PPT를 기본으로 한다.
- 레퍼런스처럼 선 중심의 깔끔한 이력서 느낌을 사용하되, PPT 화면에 맞게 넓게 구성한다.
- 흰 배경, 검정 텍스트, 얇은 회색 구분선, 아주 절제된 연한 블루 포인트를 사용한다.
- 사진 또는 아바타 영역을 반드시 포함한다.
- 정보는 짧은 문장과 키워드 중심으로 작성한다.
- 장식보다 가독성과 빠른 정보 파악을 우선한다.
- 한 페이지에 너무 많은 문단을 넣지 않는다.

## 4. 구성 요소

### 4.1 프로필 영역

#### 포함 내용

- 프로필 사진 또는 아바타
- 이름
- 희망 직무
- 연락처
- 이메일

#### 작성 기준

- 민감한 개인정보는 사용자가 제공한 경우에만 넣는다.
- 사진이 없으면 기본 아바타나 이미지 자리로 대체한다.
- 연락처가 없으면 이메일만 표시할 수 있다.

### 4.2 핵심 경험

#### 포함 내용

- 대표 프로젝트 또는 활동 2~3개
- 각 경험의 역할
- 한 줄 성과 또는 의미

#### 작성 기준

- 경험명은 짧고 구체적으로 작성한다.
- 성과 수치가 없으면 임의로 만들지 않는다.
- 지원 직무와 가장 관련 높은 경험을 우선 배치한다.

### 4.3 보유 역량

#### 포함 내용

- 문제 정의
- 데이터 구조화
- UI 설계
- 협업
- 문서화
- 그 외 사용자에게 확인된 역량

#### 작성 기준

- 역량은 3~6개 키워드로 압축한다.
- 경험에서 확인되지 않은 역량은 넣지 않는다.
- 키워드는 칩 또는 짧은 목록 형태로 표시한다.

### 4.4 기술·도구

#### 포함 내용

- 사용 기술
- 협업 도구
- 디자인 도구
- AI/API 관련 이해
- 자격 또는 교육 이력

#### 작성 기준

- 사용한 적 없는 기술은 넣지 않는다.
- 숙련도를 임의로 점수화하지 않는다.
- 도구명은 짧은 목록으로 정리한다.

### 4.5 한 줄 소개

#### 포함 내용

- 지원자의 강점과 방향성을 설명하는 한 문장

#### 작성 기준

- 과장된 수식어보다 실제 경험에서 도출된 강점을 사용한다.
- 직무와 연결되는 문장으로 작성한다.
- 1~2줄을 넘기지 않는다.

## 5. AI 작성 원칙

- 승인된 Wiki, 이력서, 활동 기록에 있는 정보만 사용한다.
- 없는 연락처, 자격증, 경력, 기술은 만들지 않는다.
- 긴 문단보다 짧은 항목과 키워드로 요약한다.
- 가장 중요한 경험과 역량만 선별한다.
- 사진이나 대표 이미지는 프로필 또는 대표 프로젝트 보조 자료로 사용한다.

## 6. 피해야 할 것

- A4 문서처럼 세로로 길게 만드는 것
- 사진 없이 텍스트만 배치하는 것
- 모든 경험을 한 장에 무리하게 넣는 것
- 근거 없는 자격증, 경력, 기술을 추가하는 것
- 화려한 장식으로 정보 구조를 흐리는 것
