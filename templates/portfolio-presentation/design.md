# 4번 PPT 발표형 디자인 기준

## 1. 템플릿 목적

PPT 발표형은 프로젝트 하나를 짧은 발표 흐름에 맞춰 설명하는 3장짜리 PPT 형식이다.

상세기술 포트폴리오처럼 깊게 설명하기보다, 발표자가 말로 보충할 수 있도록 핵심만 남긴다. 개요, 과정, 결과가 자연스럽게 이어지는 심플한 보고서형 발표 자료를 목표로 한다.

## 2. 전체 구조

PPT 발표형은 프로젝트 하나당 3장으로 구성한다.

| 페이지 | 제목 | 역할 |
|---:|---|---|
| 1 | 프로젝트 개요 | 문제, 목표, 역할을 간단히 설명한다. |
| 2 | 진행 과정 | 요구 분석부터 수정까지의 흐름과 어려움, 해결을 정리한다. |
| 3 | 결과 및 결론 | 결과, 의미, 개선 방향을 정리한다. |

## 3. 디자인 방향

- 출력 형식은 16:9 가로형 PPT를 기본으로 한다.
- 흰 배경과 진한 남색 포인트를 사용한다.
- 상단에는 얇은 남색 라인을 둔다.
- 제목은 왼쪽에 크게 배치하고, 영어 소제목을 작게 보조로 넣을 수 있다.
- 내용 영역은 연한 라벤더/블루 톤의 큰 박스를 사용한다.
- 불필요한 아이콘, 사진, 차트, 장식 카드는 넣지 않는다.
- 발표자가 말할 여백을 남기기 위해 텍스트를 짧게 유지한다.
- 한 장에는 하나의 역할만 담는다.

## 4. 페이지별 구성

### 4.1 프로젝트 개요

#### 포함 내용

- 프로젝트명
- 프로젝트 한 줄 요약
- 문제
- 목표
- 역할

#### 레이아웃 기준

- 왼쪽 상단에 `프로젝트 개요` 제목을 배치한다.
- 제목 아래에 `Project Overview`를 작게 표시한다.
- 오른쪽 상단에는 안내 문장 또는 요약 문장을 배치한다.
- 중앙에는 프로젝트 핵심 내용을 담는 넓은 요약 박스를 둔다.
- 하단에는 `문제`, `목표`, `역할` 3개만 간단히 정리한다.

### 4.2 진행 과정

#### 포함 내용

- 요구 분석
- 화면 설계
- 구현
- 수정
- 어려움
- 해결

#### 레이아웃 기준

- 왼쪽 상단에 `진행 과정` 제목을 배치한다.
- 제목 아래에 `Process Page`를 작게 표시한다.
- 중앙에는 `요구 분석 → 화면 설계 → 구현 → 수정` 흐름을 한 줄로 보여준다.
- 하단에는 `어려움`과 `해결`을 두 줄로 간단히 정리한다.
- 복잡한 상세 설명은 넣지 않고 발표자가 말로 보충하도록 남긴다.

### 4.3 결과 및 결론

#### 포함 내용

- 구현 결과
- 프로젝트의 의미
- 개선 방향
- 배운 점

#### 레이아웃 기준

- 왼쪽 상단에 `결과 및 결론` 제목을 배치한다.
- 제목 아래에 `Result Conclusion`을 작게 표시한다.
- 하단 절반에는 큰 연한 배경 박스를 둔다.
- 왼쪽에는 결론 문장을 크게 배치한다.
- 오른쪽에는 `결과`, `의미`, `개선`을 짧은 문단으로 정리한다.

## 5. AI 작성 원칙

- 프로젝트 하나를 3장 안에 설명할 수 있도록 압축한다.
- 승인된 Wiki, 활동 기록, 업로드 자료 기반으로만 작성한다.
- 없는 성과, 수치, 역할, 기술은 만들지 않는다.
- 발표자가 말할 수 있도록 문장은 짧게 작성한다.
- 상세한 기술 설명보다 개요, 과정, 결과의 흐름을 우선한다.
- 이미지나 차트는 기본적으로 사용하지 않는다. 사용자가 명시적으로 요청한 경우에만 넣는다.

## 6. 피해야 할 것

- 1번 상세기술 포트폴리오처럼 페이지를 계속 늘리는 것
- 카드, 아이콘, 사진, 차트를 과하게 넣는 것
- 한 페이지에 많은 문장을 넣어 발표 여백을 없애는 것
- 프로젝트 여러 개를 한 번에 넣는 것
- 근거 없는 성과를 발표용으로 과장하는 것
