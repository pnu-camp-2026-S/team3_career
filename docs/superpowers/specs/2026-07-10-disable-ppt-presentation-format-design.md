# PPT 발표 스펙 선택 비활성화 설계

## 목적

포트폴리오 형식 선택 화면의 `PPT 발표 스펙` 기능이 아직 구현되지 않았음을 사용자에게 알리고, 해당 형식으로 포트폴리오 생성을 시작하지 못하게 한다.

## 선택한 방식

`PPT 발표 스펙` 카드 버튼에 브라우저 기본 `disabled` 속성과 `aria-disabled="true"`를 적용한다. 카드 제목은 `PPT 발표 스펙 (미구현)`으로 변경한다.

이 방식은 JavaScript 클릭 처리에서만 차단하는 것보다 마우스와 키보드 입력을 모두 확실히 막고, 카드를 완전히 숨기는 것과 달리 향후 제공될 형식임을 사용자에게 보여준다.

## 화면과 동작

- 카드의 기존 미리보기와 설명은 유지한다.
- 제목에 `(미구현)`을 표시한다.
- 비활성 카드는 흐린 배경·글자·테두리와 `not-allowed` 커서로 구분한다.
- hover 이동, 선택 테두리와 `선택됨` 표시는 적용하지 않는다.
- JavaScript 이벤트 연결도 `disabled` 카드를 건너뛰어 이중으로 보호한다.
- 기본 선택값인 `상세 기술 포트폴리오`와 나머지 형식 선택 동작은 유지한다.

## 오류 처리

비활성 카드에서 클릭 이벤트가 발생하더라도 형식 값과 선택 상태를 변경하지 않는다. URL 편집이나 기존 저장 데이터로 `PPT 발표 스펙`이 들어오는 경우의 조회·표시 로직은 기존 데이터 호환성을 위해 삭제하지 않는다.

## 검증

- HTML에 `disabled`, `aria-disabled="true"`, `PPT 발표 스펙 (미구현)`이 있는지 확인한다.
- CSS에 비활성 카드 상태가 있는지 확인한다.
- JavaScript가 비활성 카드를 이벤트 연결 대상에서 제외하는지 확인한다.
- `npm test`와 `npm run build`를 실행한다.

## 완료 기준

1. 사용자가 `PPT 발표 스펙` 카드를 선택할 수 없다.
2. 카드에 `미구현` 상태가 명확히 보인다.
3. 다른 포트폴리오 형식은 기존처럼 선택할 수 있다.
4. 테스트와 프로덕션 빌드가 통과한다.
