# AI 자동 정리 및 분석 기준

AI는 자료를 단순 저장하지 않고 포트폴리오 작성에 활용하기 쉬운 구조로 정리해야 한다.

> **구현 현황 (2026-07-08)** — 이 기준의 1차 구현으로 **단일 파일 AI 분석 MVP**가 동작한다.
>
> - 파일 1개 업로드 → 내용 추출 → AI 분석(요약·핵심 포인트·하위 폴더 추천·역할/기술/성과 근거) → `summary.md` / `index.json` / `log.md` 생성 (`/api/analysis/single-file`, 테스트 화면 `html/analysis.html`)
> - 메인 "분석 시작" 버튼 → 저장된 모든 분석의 **종합 분석**(`/api/analysis/aggregate`) → 메인 키워드 개요 + 포트폴리오 생성 "강조할 AI 역량 키워드" 반영
> - 계획: [단일 파일 AI 분석 MVP 구현 계획](single-file-ai-analysis-mvp.md) / 파일 구성: [파일 구성 안내](single-file-analysis-files.md)
> - 아직 미구현: 프로젝트 폴더 단위 다중 파일 분석, 분석 결과 사용자 확정 저장(DB), 자기소개서·면접 소재 추천

## AI 처리 목표

1. 파일 내용 확인
2. 핵심 내용 요약
3. 프로젝트 전체 요약 수정
4. 적절한 폴더 위치 추천
5. 파일별 요약 생성
6. `index.json` 갱신
7. `log.md` 작성
8. 분석 기능에서 사용할 구조화 데이터 생성

AI가 만든 내용에는 가능한 한 근거 파일, 추출 근거, 사용자 입력 출처를 함께 표시한다.

## 분석하기 기능 기준

분석하기 버튼 활성화 조건:

1. 최소 1개 이상의 파일이 존재한다.
2. `index.json` 생성이 완료되어 있다.
3. `summary.md` 생성이 완료되어 있다.
4. 분석 대상 파일 상태가 `uploaded` 또는 `completed`이다.
5. 삭제 예정 파일이 포함되지 않는다.

현재 구현: 메인의 분석 시작 버튼은 항상 누를 수 있고, 분석된 자료가 없으면 실행하지 않고 안내 문구를 보여준다(`no_data`). 종합 분석은 `analysisStatus: completed`인 결과만 사용한다.

## 분석 결과 예시

- 활동 요약
- 프로젝트 역할 분석
- 사용 기술 분석
- 성과 및 임팩트 정리
- 포트폴리오 문장 추천
- 자기소개서 소재 추천
- 면접 답변 소재 추천
