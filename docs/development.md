# 개발 우선순위와 코드 작업 규칙

## 개발 우선순위

1차 개발:

- 파일 업로드
- 파일 목록 조회
- 파일 삭제
- 파일 미리 보기
- 폴더 생성
- 폴더별 파일 저장
- 같은 폴더 안 파일 드래그 앤 드롭 정렬
- 파일 순서 저장
- 포트폴리오 생성 화면
- 포트폴리오 관리 화면
- 활동 추천 화면
- 마이페이지

2차 개발:

- Supabase Auth 연결
- Supabase DB 테이블 설계
- 업로드 자료 기반 경험 목록 저장
- 승인된 Wiki 저장
- 포트폴리오 초안/완성본 저장
- 활동 추천 저장
- AI API를 Next.js API Route 또는 Server Action 기준으로 정리

3차 개발:

- 실제 GitHub OAuth 연결
- 실제 GitHub 파일 동기화
- 분석 결과 버전 관리
- Vercel 배포 운영 설정

## 코드 작업 규칙

1. 기존 화면 이름과 사용자 흐름을 우선 유지한다.
2. Next.js 전환 단계에서는 기존 HTML/CSS/JS와 localStorage 기반 동작을 유지한다.
3. 활성 HTML 화면은 `html/`, 스타일은 `css/`, 브라우저 스크립트는 `js/`에 유지한다.
4. Next.js 라우팅은 `app/`에서 담당한다.
5. 기존 정적 자산은 `/css`, `/js`, `/*.html` 경로가 깨지지 않게 제공한다.
6. DB 연동은 Next 전환 이후 페이지별 이슈와 PR로 진행한다.
7. 새 기능을 추가하면 관련 정적 구조 테스트도 함께 갱신한다.
8. 사용자가 승인하지 않은 AI 결과를 최종 데이터처럼 저장하지 않는다.

## 작업 완료 전 확인 목록

- `npm test`가 통과하는가?
- `npm run build`가 통과하는가?
- 새 화면 문구가 한국어인가?
- 개인정보나 API 키가 노출되지 않았는가?
- 기존 `.html` 경로 접속이 유지되는가?
- `/css`, `/js` 정적 자산 경로가 유지되는가?
- Supabase DB 연동이 필요한 기능을 이번 전환 PR에 섞지 않았는가?
