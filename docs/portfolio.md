# 포트폴리오 생성 및 관리 기준

## 포트폴리오 생성 기준

포트폴리오 생성 페이지는 사용자가 저장한 경험 데이터를 바탕으로 목표 직무에 맞는 포트폴리오 초안을 생성하는 화면이다.

기본 흐름:

```text
경험 선택
→ 키워드 확인 및 수정
→ 포트폴리오 형식 선택
→ 생성
→ 결과 확인
→ AI 수정안 확인
→ 사용자 적용
→ PPT 다운로드 또는 저장
```

구현 기준:

- 한 화면 안에서 간단하게 사용할 수 있게 구성한다.
- 초기 구현에서는 검색, 필터링, 이전으로 돌아가기, 복잡한 임시저장 기능을 줄인다.
- 생성 전에는 예시 포트폴리오를 보여주고, 생성 후에는 실제 결과물로 교체한다.
- 포트폴리오 형식은 4가지 정도로 단순하게 제시한다.
- 키워드는 AI 추천 키워드와 사용자 입력 키워드를 구분한다.
- 키워드에는 가능하면 추출 근거와 직무 관련성을 함께 보여준다.
- **AI 추천 키워드의 근거는 사용자가 승인·수정한 요약이다.** 선택한 경험의 파일 `summary.md`가 `POST /api/portfolio/keywords`의 입력이 되고, 프로젝트/메인 종합의 `portfolioKeywords`도 강조 키워드 소스로 함께 쓴다. 포트폴리오 본문 생성 프롬프트는 이 키워드를 슬롯인해 사용하므로, 근거 없는 성과·수치는 만들지 않는다.
- 생성 중에는 로딩 화면과 진행 상태를 보여준다.
- 최종 결과물은 PPT 다운로드 또는 저장을 우선한다.
- PPT에 들어가기 좋은 짧고 명확한 문장 길이를 고려한다.
- 정보가 부족하면 부족한 정보를 안내한다.

## 포트폴리오 관리 기준

포트폴리오 관리 페이지는 완료 포트폴리오와 임시저장 포트폴리오를 한곳에서 관리하는 화면이다.

목록 기준:

- 좋아요한 포트폴리오 우선
- 그다음 최신순
- 별도의 좋아요 모아보기 버튼은 만들지 않는다.
- 싫어요 기능은 추가하지 않는다.
- 좋아요, 열기, 수정, ppt 저장, 삭제 5가지 버튼을 가진다
- 상단 필터 기능은 추가하지 않는다
- 임시저장 기능은 추가하지 않는다

카드 표시 정보:

- 제목
- 생성 날짜
- 사용 목적
- 간단 요약
- 상태

완료 포트폴리오 기능:

- 열기: 사이트 안에서 웹 미리보기
- PPT 저장: 먼저 `/api/portfolios`로 보관함 DB 저장을 완료한 뒤, 저장된 포트폴리오 데이터를 기준으로 `.pptx` 파일을 다운로드한다. DB 저장이 실패하면 다운로드를 중단하고 실패 상태를 보여준다.
- 좋아요: 누른 항목을 목록 상단에 우선 정렬
- 수정: 포트폴리오 생성/편집 화면으로 이동해 이어서 작성한다.
- 삭제: 선택한 포트폴리오를 삭제한다
