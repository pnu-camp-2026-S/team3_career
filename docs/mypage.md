# 마이페이지 및 프로필 데이터 기준

마이페이지는 추천, AI 분석, 포트폴리오 생성에 필요한 사용자 기본 데이터를 관리하는 곳이다.

사용자별 프로필 데이터의 구조와 저장 위치는 [데이터 구조 및 API 기준](data-model.md)의 `user_profiles` 항목을 함께 참고한다.

## 필수 영역

- 기본 정보: 프로필 사진, 이름, 성별, 생년월일
- 연락처 정보: 이메일, 휴대폰번호, 주소
- 학력: 학교 구분, 학교명, 재학기간, 전공, 부전공/연계전공, 학점
- 희망근무조건: 희망 직무, 세부직무, 업종, 지역, 연봉
- 관심분야: 관심 분야, 관심 기업, 관심 산업

## 데이터 관리 기준

- 마이페이지 입력 데이터는 Supabase `public.user_profiles` 테이블에 저장하고 조회한다. 화면은 `/api/profile`(GET/PUT)로 접근한다.
- 로그인 사용자별 데이터는 Supabase Auth 사용자 ID(`user_id`)를 기준으로 구분한다.
- 저장 대상은 사용자가 직접 입력하거나 수정한 기본 정보, 연락처 정보(`name`, `gender`, `birth_date`, `email`, `phone`, `address`), 학력(`educations`), 희망근무조건(`preferences`), 관심분야(`chips`)이다.
- 화면의 임시 상태는 `localStorage`(`myfitfolioProfile`)로 캐시하지만, 최종 저장 데이터 기준은 Supabase다. 서버 요청이 실패하면 로컬 캐시로 폴백한다.
- 저장된 값이 비어 있으면 소셜 로그인 메타데이터로 기본값을 채운다(이름, 이메일 등).
- 저장 성공, 저장 실패, 불러오기 실패 상태를 화면에 한국어로 안내한다.
- 개인정보가 포함되므로 Supabase RLS 정책(`auth.uid() = user_id`)으로 사용자별 데이터 접근 권한을 분리한다.

## 직무 및 관심분야 선택 기준

- 희망 직무와 관심분야는 chip 형태로 선택/해제할 수 있어야 한다.
- 희망 직무가 바뀌면 더 이상 맞지 않는 세부직무 선택값은 초기화한다.
