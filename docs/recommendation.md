# 활동 추천 기준

활동 추천은 사용자의 기본 정보, 경험, 희망 직무, 학과 정보를 기반으로 부족한 부분을 보완하는 방향이어야 한다.

## 추천 유형

- 전체
- 공모전
- 대외활동
- 자격증
- 교육

## 활동 카드 필수 표시 정보

- 활동명
- 활동 유형
- 모집 기간
- 활동 기간
- 주최/운영 기관
- 추천 이유
- 마감일
- 원본 사이트 링크
- 저장 여부
- 상세 정보 보기 버튼

## 추천 원칙

- 랜덤 추천을 하지 않는다.
- 사용자의 현재 상태와 목표 직무 사이의 부족한 부분을 채우는 활동을 우선 추천한다.
- `추천 이유`를 카드 안에 표시해 사용자가 납득할 수 있게 한다.
- 상세 정보 보기 클릭 시 상세 영역은 카드 아래에 펼쳐지고, 기존 리스트 순서를 강제로 바꾸지 않는다.
- 활동 저장 시 서비스 내 캘린더에 반영한다.
- Google Calendar 저장은 프로토타입 단계에서 실제 연동하지 않고, 이후 확장을 위한 버튼만 제공한다.
- Google Calendar 연동 버튼을 클릭하면 실제 인증 화면으로 이동하지 않고, 임시 아이디 기반 프로토타입이라는 안내를 보여준다.
- 같은 날짜에 이미 저장된 활동이 있으면 `이미 활동이 있습니다. 추가하시겠습니까?` 확인 팝업을 보여준다.
