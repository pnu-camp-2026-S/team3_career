# 자료 입력 및 파일관리 기준

자료 입력 페이지의 목적은 사용자가 프로젝트별 자료를 업로드하고, AI가 요약·분류·인덱싱할 수 있도록 정리하는 것이다.

## 지원해야 할 입력 방식

- 버튼 클릭 업로드
- 드래그 앤 드롭 업로드
- AI 대화창 기반 Markdown 파일 생성
- GitHub 연동 확장용 버튼과 임시 동기화 상태 표시

## 지원 파일 예시

```text
.pdf
.ppt
.pptx
.doc
.docx
.md
.txt
.csv
.xlsx
.png
.jpg
.jpeg
```

## 파일 업로드 후 필요한 처리

1. 파일 저장
2. 파일 메타데이터 생성
3. AI 분석 및 파일 내용 추출 즉시 시작
4. 파일별 핵심 내용 요약 생성
5. `summary.md` 갱신
6. `index.json` 갱신
7. `log.md` 갱신
8. 분석 결과 상태 저장

## 프로젝트 폴더와 세부 폴더 기준

- 메인 페이지의 프로젝트 폴더와 파일 관리 탭의 프로젝트 폴더는 같은 데이터를 사용해야 한다.
- 메인 페이지에서는 프로젝트 폴더 목록을 간단히 보여 주고, 파일 관리 탭에서는 같은 프로젝트 폴더의 세부 폴더와 자료 목록을 자세히 보여 준다.
- 메인 페이지에서 프로젝트 폴더를 클릭하면 파일 관리 탭으로 이동하고, 선택한 프로젝트가 열린 상태로 보여야 한다.
- 파일 관리 탭에서는 프로젝트별로 폴더를 생성하고, 각 프로젝트 폴더 안에 자료를 추가하는 방식을 사용한다.
- 프로젝트 폴더 안에는 `발표자료`, `보고서`, `데이터`, `코드`, `기획서`, `제출 자료`처럼 자료 성격에 따른 세부 폴더를 만들 수 있어야 한다.
- 세부 폴더를 클릭하면 해당 세부 폴더에 들어 있는 자료만 열려서 보여야 한다.
- 파일은 반드시 하나의 프로젝트 폴더에 속해야 하며, 가능하면 하나의 세부 폴더에도 속하도록 관리한다.
- 프로젝트 폴더, 세부 폴더, 파일 목록은 동기화 가능한 하나의 데이터 구조로 관리한다.

## 파일 순서와 우선순위

- 파일 순서는 반드시 백엔드 또는 영속 저장소에 저장한다.
- 파일 순서는 포트폴리오 작성 우선순위로 사용한다.
- `order`는 화면 표시 순서이고, `priority`는 AI 분석과 포트폴리오 생성 우선순위이다.

## 삭제와 휴지통 기준

- 삭제는 우선 soft delete 방식으로 처리하고, 삭제된 파일은 휴지통에서 확인할 수 있어야 한다.
- 휴지통에는 삭제된 파일명, 원래 프로젝트 폴더, 원래 세부 폴더, 삭제 시각, 삭제한 사용자, 분석 상태를 표시한다.
- 휴지통의 파일은 복구하거나 영구 삭제할 수 있어야 한다.
- 복구 시 파일은 원래 프로젝트 폴더와 세부 폴더 위치로 돌아가야 한다.
- 원래 폴더가 사라진 경우에는 사용자가 복구 위치를 다시 선택할 수 있게 한다.
- 영구 삭제는 사용자 확인 팝업을 거친 뒤 처리한다.
- 휴지통에 있는 파일은 AI 분석, `summary.md` 갱신, 포트폴리오 생성, 활동 추천 근거에서 제외한다.
- 파일 삭제, 복구, 영구 삭제 이력은 `log.md`에 기록한다.

## 저장 파일 기준

- 폴더 내부 파일 목록은 `index.json`에 저장한다.
- 전체 요약은 `summary.md`에 저장한다.
- 변경 이력은 `log.md`에 저장한다.
- 자료가 없는 경우 AI 대화로 `activity-summary.md` 같은 Markdown 파일을 생성해 저장할 수 있어야 한다.

## 상태값 기준

```text
uploading: 업로드 중
uploaded: 업로드 완료
failed: 업로드 실패
deleted: 삭제됨

pending: 분석 시작 전 임시 상태 또는 재시도 대기
analyzing: 분석 중
completed: 분석 완료
failed: 분석 실패

draft: 임시 저장
saved: 저장 완료
sync_pending: GitHub 동기화 대기
synced: GitHub 동기화 완료
sync_failed: GitHub 동기화 실패
```
