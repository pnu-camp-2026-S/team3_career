# GitHub 임시 연동 기준

프로토타입 단계에서는 실제 GitHub OAuth, GitHub API 호출, 원격 커밋 동기화를 구현하지 않는다.  
대신 이후 확장을 고려해 마이페이지의 GitHub 아이디 연결 버튼과 프로젝트별 레포지토리 연결 버튼만 제공한다.

## 마이페이지 GitHub 연결 기준

- 마이페이지에서 GitHub 아이디를 입력하거나 확인할 수 있어야 한다.
- `GitHub 연결하기` 버튼을 제공한다.
- 버튼 클릭 시 실제 GitHub 인증 화면으로 이동하지 않는다.
- 버튼 클릭 시 `현재 프로토타입에서는 GitHub 실제 연동 없이 임시 연결 상태만 저장합니다.` 같은 안내를 보여준다.
- 프로토타입 단계에서는 모의 로그인 사용자 ID를 기준으로 GitHub 임시 연결 상태를 저장한다.

## 프로젝트별 GitHub 레포지토리 연결 기준

- 파일 관리 탭에서 프로젝트 폴더별로 GitHub 레포지토리 연결 버튼을 제공한다.
- 프로젝트별로 저장소 소유자, 저장소 이름, 브랜치, 기준 경로를 입력하거나 표시할 수 있어야 한다.
- 버튼 클릭 시 실제 GitHub API를 호출하지 않고 임시 연결 상태만 저장한다.
- 프로젝트 폴더 각각이 서로 다른 GitHub 레포지토리 또는 경로를 가질 수 있는 구조로 설계한다.
- 세부 폴더 단위 연결은 이후 확장 가능하도록 고려하되, 프로토타입에서는 프로젝트 폴더 단위 연결을 우선한다.

## GitHub 연동 시 필요한 정보

- GitHub 아이디
- 저장소 소유자
- 저장소 이름
- 브랜치
- 기준 경로
- 동기화 사용 여부
- 마지막 동기화 시각

## 프로토타입 동작

1. GitHub 연결 버튼 클릭
2. 실제 OAuth 없이 임시 연결 안내 표시
3. 사용자가 입력한 GitHub 아이디 또는 레포지토리 정보를 저장
4. 화면에 `연결됨`, `미연결`, `동기화 준비 중` 같은 상태 표시
5. `summary.md`, `index.json`, `log.md`, 원본 파일 동기화는 실제 수행하지 않고 이후 확장 예정 상태로 표시
6. 변경 내역 커밋은 실제 수행하지 않고 버튼 또는 상태 표시만 제공
