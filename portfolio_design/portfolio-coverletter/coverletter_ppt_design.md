# 자기소개서형 포트폴리오 PPT 디자인 기준

## 1. 목적

자기소개서형 포트폴리오는 사용자의 승인된 경험 자료를 바탕으로, 자기소개서 문항의 흐름을 PPT에서 읽기 쉽게 보여주는 지원용 포트폴리오다.

이 형식은 기존 A4 Word 자기소개서가 아니라, PptxGenJS로 생성할 수 있는 16:9 발표/제출 겸용 PPT를 기준으로 한다. 생성 결과는 `portfolio_design/portfolio-coverletter/coverletter_ppt_template.json`의 좌표와 토큰을 우선 따른다.

## 2. 참고한 일반 구성

- 이력서, 커버레터, 포트폴리오는 지원 직무에 맞춰 조정되어야 하며 지원자의 역량과 경험을 근거로 보여줘야 한다.
- 커버레터형 포트폴리오는 기술과 경험, 지원 동기, 직무에 대한 관심과 적합성을 요약하는 데 적합하다.
- 자기소개 PPT 템플릿은 보통 지원자 소개, 지원 동기, 주요 경험/포트폴리오, 핵심 보유 역량, 입사 후 포부 순서로 구성된다.
- PptxGenJS는 슬라이드, 도형, 텍스트, 테마, 커스텀 레이아웃을 코드로 생성할 수 있으므로 고정 좌표 JSON 템플릿과 잘 맞는다.

## 3. 전체 슬라이드 구조

| 순서 | 슬라이드 | 역할 |
|---:|---|---|
| 1 | 표지 | 지원자명, 지원 직무, 핵심 한 줄 소개 |
| 2 | 지원 동기 | 직무 관심이 생긴 계기와 경험 근거 |
| 3 | 직무 적합 역량 | 핵심 역량 3개와 각 근거 경험 |
| 4 | 대표 경험 서술 | 문제, 행동, 결과/배운 점을 자기소개서 문장 흐름으로 정리 |
| 5 | 문제 해결 및 협업 방식 | 사용자의 일하는 방식과 협업 태도 |
| 6 | 입사 후 기여 계획 | 초기 적응, 실무 기여, 성장 방향 |
| 7 | 자기소개서 문항 연결 | 문항별 답변 요약과 보완 필요 정보 |

## 4. 디자인 방향

- 화면비율은 16:9 와이드(`LAYOUT_WIDE`)를 기본으로 한다.
- 배경은 `#F6F8FC`, 본문 카드는 `#FFFFFF`, 주요 강조색은 `#5965E8`을 사용한다.
- 폰트는 `Malgun Gothic`을 기본으로 한다.
- 문서는 과한 장식보다 채용 담당자가 빠르게 훑을 수 있는 문서형 레이아웃을 우선한다.
- 각 슬라이드는 좌측에 문항/섹션 제목, 우측에 근거 경험이나 답변 요약을 배치하는 2단 구조를 기본으로 한다.
- 텍스트는 한 슬라이드에 3~5개 포인트만 배치한다.
- 사용자의 실제 자료에 없는 회사명, 성과 수치, 역할, 자격 취득 여부는 만들지 않는다.

## 5. PptxGenJS 구현 기준

- 좌표 단위는 inch다.
- 슬라이드 크기는 `13.333 x 7.5`로 본다.
- 모든 슬라이드는 같은 top bar, page number, footer rule을 사용한다.
- 제목은 `addText`, 카드와 라인은 `addShape`, 강조 chip은 `roundRect`로 구현한다.
- 이미지가 없을 때는 사진 placeholder를 넣지 않고 이름 이니셜 원형 배지를 사용한다.
- 텍스트가 길면 자르기보다 JSON의 `maxChars` 기준에 맞춰 OpenAI가 짧게 생성하게 한다.

## 6. OpenAI 생성 규칙

OpenAI는 다음 JSON 구조를 생성한다.

```json
{
  "format": "자기소개서형 포트폴리오",
  "title": "지원자명 | 지원 직무",
  "headline": "경험 기반 핵심 한 줄 소개",
  "applicant": {
    "name": "",
    "targetRole": "",
    "major": "",
    "email": "",
    "phone": ""
  },
  "slides": [
    {
      "id": "motivation",
      "title": "지원 동기",
      "subtitle": "",
      "narrative": "",
      "evidence": [
        { "label": "근거 경험", "text": "" }
      ]
    }
  ],
  "missingFields": []
}
```

단, 렌더링 좌표와 색상은 `coverletter_ppt_template.json`을 따르고, OpenAI는 해당 JSON의 `bindings`에 들어갈 텍스트만 생성한다.

## 7. 슬라이드별 작성 기준

### 7.1 표지

- 지원자 이름과 지원 직무를 가장 크게 보여준다.
- 핵심 한 줄 소개는 사용자의 경험과 직무가 연결되는 문장으로 작성한다.
- 하단에는 전공, 주요 키워드, 선택 경험 개수를 chip 형태로 배치한다.

### 7.2 지원 동기

- 막연한 관심이 아니라 어떤 경험 때문에 해당 직무에 관심을 갖게 되었는지 쓴다.
- 회사명이 없으면 특정 회사명을 만들지 않고 직무 중심으로 작성한다.
- 오른쪽 근거 카드에는 연결 경험 2개를 짧게 보여준다.

### 7.3 직무 적합 역량

- 역량은 3개만 제시한다.
- 각 역량은 `역량명 + 행동 근거 + 결과/배운 점` 구조를 따른다.
- 너무 포괄적인 "성실함", "열정"보다 경험에서 확인 가능한 행동형 역량을 우선한다.

### 7.4 대표 경험 서술

- 자기소개서 문항 답변처럼 자연스러운 문단형 흐름을 사용한다.
- 구조는 `문제 상황 → 나의 행동 → 결과/배운 점 → 직무 연결`이다.
- 정량 결과가 없으면 만들지 않고 정성적 변화나 배운 점만 쓴다.

### 7.5 문제 해결 및 협업 방식

- 사용자가 문제를 정의하고 협업하거나 실행한 방식을 보여준다.
- 협업 경험이 없으면 무리하게 팀 활동을 만들지 않고 개인 실행 방식으로 정리한다.

### 7.6 입사 후 기여 계획

- `초기 적응`, `실무 기여`, `성장 방향` 3단계로 쓴다.
- 기업 정보가 없으면 직무에서의 기여 방향 중심으로 작성한다.

### 7.7 자기소개서 문항 연결

- 자소서 문항 3~4개와 각 문항에 연결할 경험을 매핑한다.
- 보완할 정보가 있으면 `missingFields`에 적는다.

## 8. 금지 사항

- 확인되지 않은 회사명, 성과 수치, 수상, 자격 취득 여부를 만들지 않는다.
- 화면에 긴 자기소개서 전문을 그대로 붙이지 않는다.
- 모든 슬라이드를 동일한 문장 구조로 반복하지 않는다.
- 사진, 아이콘, 그래프가 없는데 있는 것처럼 표현하지 않는다.
- PPT 템플릿 JSON 좌표를 무시하고 임의의 레이아웃을 만들지 않는다.
