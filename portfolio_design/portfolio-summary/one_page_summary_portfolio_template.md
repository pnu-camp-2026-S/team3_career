# 작성 체크리스트

- [ ] 이름, 희망 직무, 이메일 등 기본 정보가 정확한가?
- [ ] 연락처 등 민감한 개인정보는 사용자가 제공한 경우에만 넣었는가?
- [ ] 핵심 경험은 2–3개만 선별했는가?
- [ ] 각 경험은 역할과 한 줄 성과/의미로 정리했는가?
- [ ] 성과 수치, 자격증, 기술, 경력 등을 임의로 만들지 않았는가?
- [ ] 보유 역량은 실제 경험에서 확인된 3–6개 키워드인가?
- [ ] 기술·도구·교육 항목은 사용 또는 이수 사실이 확인된 것만 포함했는가?
- [ ] 한 줄 소개가 희망 직무와 연결되는가?

# PPT 배치 가이드

| PPT 영역 | Markdown 섹션 | 작성 방식 |
|---|---|---|
| 상단 | 제목 영역 | 포트폴리오 라벨, 이름 기반 제목, 희망 직무, 페이지 번호 |
| 좌측 상단 | PROFILE | 사진/아바타, 이름, 직무, 연락처, 이메일 |
| 우측 상단 | KEY EXPERIENCE | 대표 경험 2–3개, 역할, 한 줄 성과/의미 |
| 좌측 하단 | SKILLS | 3–6개 역량 키워드 |
| 우측 하단 | TOOLS / EDUCATION | 기술, 협업 도구, 디자인 도구, 자격·교육 |
| 하단 | 한 줄 소개 | 강점과 직무 방향성을 담은 1문장 |

```md
# PORTFOLIO SUMMARY

## 홍길동 1페이지 요약

**UX/UI 기획자 · 서비스 개선 경험과 데이터 기반 문제 정의 중심**  
**PAGE 01**

## PROFILE · 기본 정보

| 항목 | 내용 |
|---|---|
| 사진 또는 아바타 | profile.png |
| 이름 | 홍길동 |
| 희망 직무 | UX/UI 기획자 |
| 요약 프로필 | 사용자 문제를 구조화하고 서비스 흐름을 개선하는 기획자 |
| 연락처 | 010-0000-0000 |
| 이메일 | hello@example.com |

## KEY EXPERIENCE · 핵심 경험 2–3개

### 경험 01 · 모바일 앱 온보딩 개선 프로젝트

- **역할:** 사용자 리서치, 화면 흐름 설계, 와이어프레임 작성
- **성과/의미:** 신규 사용자의 초기 진입 흐름을 단순화한 프로젝트

## SKILLS · 보유 역량 키워드

`문제 정의` `사용자 흐름 설계` `UI 구조화` `협업` `문서화`

## TOOLS / EDUCATION · 기술·도구 또는 자격·교육

### 사용 기술

- Figma
- Notion
- Google Analytics

## 한 줄 소개

> 사용자 문제를 구조화하고 실행 가능한 화면 설계로 연결하는 UX/UI 기획자 지원자입니다.
```

> 위 예시는 형식 참고용이다. 실제 파일 작성 시 확인되지 않은 개인정보, 수치, 기술, 자격은 삭제한다.
