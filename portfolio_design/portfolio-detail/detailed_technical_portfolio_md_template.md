# 6. 최종 점검 체크리스트

## 내용 점검

- [ ] 프로젝트별로 문제, 과정, 기여, 결과가 자연스럽게 이어지는가?
- [ ] 본인 기여와 팀 전체 성과가 구분되어 있는가?
- [ ] 없는 성과, 수치, 역할, 기술을 만들지 않았는가?
- [ ] 기술 스택만 나열하지 않고 실제 구현 흐름을 설명했는가?
- [ ] 결과와 회고가 다음 개선 방향으로 연결되는가?

## 시각 자료 점검

- [ ] 프로젝트 대표 이미지가 포함되어 있는가?
- [ ] 과정 설명에 필요한 화면, 구조도, 흐름도, 목업이 포함되어 있는가?
- [ ] 결과 화면 또는 최종 산출물 이미지가 포함되어 있는가?
- [ ] 이미지가 장식이 아니라 설명 근거로 사용되고 있는가?

## 문장 점검

- [ ] 한 페이지 또는 한 섹션에 하나의 핵심 메시지만 담았는가?
- [ ] 긴 문단보다 제목, 짧은 설명, 표, 카드형 문장을 사용했는가?
- [ ] 회고가 감상문이 아니라 개선 방향과 연결되는가?
