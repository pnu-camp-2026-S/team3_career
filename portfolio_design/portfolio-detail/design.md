# 상세기술 포트폴리오 A4 2장 디자인 기준

## 1. 템플릿 목적

상세기술 포트폴리오는 사용자의 프로젝트 경험을 `과정`, `기여도`, `결과` 중심으로 압축해 보여주는 A4 문서형 포트폴리오이다.

이 템플릿은 지원자의 전체 프로필과 대표 경험 3개를 2장 안에 고정 배치한다. 디자인과 공간 구조는 항상 동일하게 유지하고, AI는 사용자 정보와 경험별 내용만 바꿔 채운다.

## 2. 출력 형식

- 출력 형식은 A4 세로형 2장이다.
- PPTX 생성은 PptxGenJS 기반으로 하되, 슬라이드 크기는 A4 세로 비율에 맞춘다.
- 각 장은 위쪽 절반과 아래쪽 절반으로 나눈다.
- 각 절반 영역은 하나의 독립된 블록처럼 사용한다.
- 전체 페이지 수는 반드시 2장으로 고정한다.
- 경험은 최대 3개까지 표시한다.
- 경험이 3개보다 적으면 비어 있는 경험 영역에는 `추가 경험 입력 필요` 상태를 표시한다.

## 3. 전체 구성

| 페이지 | 위치 | 구성 | 역할 |
|---|---|---|---|
| 1장 | 위쪽 절반 | 사용자 개요 | 프로필 사진, 이름, 성별, 학력, 연락처, 희망 직무, 강조 키워드 3개를 보여준다. |
| 1장 | 아래쪽 절반 | 경험 1 | 대표 프로젝트 1개의 과정, 기여도, 결과와 성장 포인트를 보여준다. |
| 2장 | 위쪽 절반 | 경험 2 | 두 번째 프로젝트의 과정, 기여도, 결과와 성장 포인트를 보여준다. |
| 2장 | 아래쪽 절반 | 경험 3 | 세 번째 프로젝트의 과정, 기여도, 결과와 성장 포인트를 보여준다. |

## 4. 공통 디자인 방향

- 배경은 흰색을 기본으로 한다.
- 주요 카드 배경은 연한 블루 계열을 사용한다.
- 텍스트는 검정과 짙은 회색을 기본으로 하고, 강조 색상은 블루만 제한적으로 사용한다.
- 장식보다 정보 구조를 우선한다.
- 모든 영역은 여백을 충분히 두고, 글자가 카드 밖으로 넘치지 않게 작성한다.
- 둥근 네모 카드의 모서리는 부드럽게 처리하되 과하게 둥글게 만들지 않는다.
- 각 경험 영역은 같은 레이아웃을 반복한다.
- 경험별 색상, 카드 위치, 카드 개수, 화살표 위치는 바꾸지 않는다.
- 내용이 길면 줄이기보다 핵심 문장으로 압축한다.

## 5. 1장 위쪽 절반: 사용자 개요

### 포함 내용

- 프로필 사진
- 이름
- 성별
- 학력
- 전공
- 희망 직무
- 이메일 또는 연락처
- AI가 분석한 포트폴리오 강조 키워드 3개

### 레이아웃 기준

- 왼쪽에는 프로필 사진을 배치한다.
- 프로필 사진은 원형 또는 부드러운 사각형 프레임으로 표시한다.
- 오른쪽에는 이름을 가장 크게 배치한다.
- 이름 아래에는 성별, 학력, 전공, 희망 직무, 연락처를 짧은 라벨-값 형태로 정리한다.
- 사용자 개요 영역 하단에는 강조 키워드 3개를 가로로 배치한다.
- 강조 키워드는 AI 분석 우선순위가 높은 순서대로 왼쪽에서 오른쪽으로 배치한다.
- 키워드 카드는 `1순위`, `2순위`, `3순위`가 보이게 구성한다.
- 키워드는 단어 또는 짧은 구로 작성한다.

### 작성 기준

- 프로필 정보는 사용자가 입력한 정보만 사용한다.
- 성별, 학력, 전공, 연락처가 없으면 임의로 만들지 않고 `정보 없음` 또는 빈 값으로 둔다.
- 강조 키워드는 프로젝트 경험과 업로드 자료에서 근거가 확인되는 역량만 사용한다.
- 키워드는 3개를 넘기지 않는다.

## 6. 경험 영역 공통 구조

경험 1, 경험 2, 경험 3은 모두 같은 형식을 사용한다.

### 포함 내용

- 경험 번호
- 프로젝트명
- 프로젝트 한 줄 설명
- 과정 카드
- 기여도 카드
- 결과 카드
- 카드 사이 화살표
- 프로젝트를 통해 발전한 점 한 줄

### 레이아웃 기준

- 각 경험 영역 상단에는 `경험 1`, `경험 2`, `경험 3` 라벨을 둔다.
- 경험 라벨 옆 또는 아래에 프로젝트명을 배치한다.
- 프로젝트명 아래에는 프로젝트를 설명하는 한 줄 요약을 둔다.
- 본문 중앙에는 둥근 네모 카드 3개를 가로로 배치한다.
- 카드 순서는 항상 `과정` -> `기여도` -> `결과`이다.
- 각 카드 사이에는 작은 `>` 화살표를 배치한다.
- 화살표는 카드보다 작고, 중앙 높이에 맞춘다.
- 카드 3개는 같은 너비와 높이를 가진다.
- 카드 아래에는 `이 프로젝트를 통해 발전한 점`을 한 줄로 중앙 정렬한다.

### 카드 구성

| 카드 | 제목 | 영문 보조 라벨 | 내용 |
|---|---|---|---|
| 1 | 과정 | PROCESS | 프로젝트가 어떤 흐름으로 진행되었는지 2~3개 불릿으로 정리한다. |
| 2 | 기여도 | CONTRIBUTION | 사용자가 맡은 역할과 직접 수행한 일을 2~3개 불릿으로 정리한다. |
| 3 | 결과 | RESULT | 산출물, 개선 효과, 배운 점 또는 확인된 결과를 2~3개 불릿으로 정리한다. |

### 카드 디자인 기준

- 각 카드는 연한 블루 배경을 사용한다.
- 카드 테두리는 중간 밝기의 블루 또는 그레이 블루를 사용한다.
- 카드 상단에는 한국어 제목을 굵게 표시한다.
- 한국어 제목 아래에는 영문 보조 라벨을 작게 표시한다.
- 카드 본문은 불릿 목록으로 작성한다.
- 각 카드의 불릿은 최대 3개까지 사용한다.
- 한 불릿은 한 줄 또는 짧은 두 줄 안에 들어오도록 작성한다.
- 결과 카드에는 근거 없는 수치나 성과를 만들지 않는다.

## 7. 경험 1 배치

경험 1은 1장 아래쪽 절반에 배치한다.

- 1장 위쪽의 사용자 개요와 시각적으로 구분되도록 얇은 구분선 또는 충분한 여백을 둔다.
- 경험 1은 사용자가 선택한 대표성이 가장 높은 프로젝트를 사용한다.
- 대표성 기준은 희망 직무 관련성, 본인 기여도, 결과 명확성, 업로드 자료의 충분성 순서로 판단한다.
- 경험 1 영역의 구조는 `프로젝트명 -> 한 줄 설명 -> 과정/기여도/결과 카드 -> 발전한 점` 순서로 고정한다.

## 8. 경험 2와 경험 3 배치

경험 2와 경험 3은 2장에 배치한다.

- 경험 2는 2장 위쪽 절반에 배치한다.
- 경험 3은 2장 아래쪽 절반에 배치한다.
- 두 경험 사이에는 얇은 구분선 또는 충분한 여백을 둔다.
- 경험 2와 경험 3의 카드 구조, 카드 크기, 화살표 위치, 발전한 점 위치는 경험 1과 동일해야 한다.
- 경험이 부족하면 해당 영역의 프로젝트명에 `추가 경험 입력 필요`를 표시하고, 카드에는 부족한 정보 항목을 안내한다.

## 9. AI 작성 원칙

- 업로드 자료, 승인된 Wiki, 사용자가 입력한 활동 기록처럼 확인 가능한 근거만 사용한다.
- 없는 프로젝트명, 역할, 기술, 성과, 수치를 만들지 않는다.
- 본인 기여와 팀 전체 결과를 구분한다.
- 과정은 실제 진행 흐름 중심으로 작성한다.
- 기여도는 사용자가 직접 수행한 행동 중심으로 작성한다.
- 결과는 확인된 산출물과 변화 중심으로 작성한다.
- 수치가 없으면 임의 수치를 만들지 않고 정성적 결과로 표현한다.
- 발전한 점은 프로젝트 경험을 통해 사용자의 역량이 어떻게 성장했는지 한 문장으로 작성한다.
- 모든 문장은 A4 절반 영역에 들어갈 수 있도록 짧고 명확하게 쓴다.

## 11. PptxGenJS 구현 메모

- A4 세로 비율 기준으로 페이지를 만든다.
- 각 페이지는 `topHalf`와 `bottomHalf` 좌표 영역을 가진다.
- 1장 `topHalf`는 프로필 전용 영역이다.
- 1장 `bottomHalf`, 2장 `topHalf`, 2장 `bottomHalf`는 동일한 경험 컴포넌트를 재사용한다.
- 경험 컴포넌트는 카드 3개와 화살표 2개를 고정 좌표로 배치한다.
- 카드 텍스트가 길면 PptxGenJS의 `fit: "shrink"` 또는 글자 크기 축소 규칙을 적용한다.
- 카드 안의 불릿은 최대 3개까지만 렌더링한다.
- 키워드 카드는 반드시 rank 순서로 정렬해서 렌더링한다.
- 이미지가 없으면 프로필 사진 영역에는 기본 placeholder를 표시한다.

## 12. 피해야 할 것

- 경험마다 다른 레이아웃을 적용하는 것
- 한 경험을 A4 절반보다 크게 확장하는 것
- 카드 개수를 늘리거나 순서를 바꾸는 것
- 화살표 없이 카드만 나열하는 것
- 프로젝트 설명을 긴 문단으로 작성하는 것
- 근거 없는 성과 수치나 역할을 만들어내는 것
- 강조 키워드를 3개보다 많이 보여주는 것
- 두 번째 장에 사용자 프로필을 반복해서 넣는 것
