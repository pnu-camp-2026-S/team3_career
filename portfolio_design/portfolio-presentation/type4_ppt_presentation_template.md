# PPT 발표 스펙 템플릿

## 1. 템플릿 목적

이 템플릿은 지원자의 기본 정보와 프로젝트 경험을 발표용 포트폴리오로 구성하기 위한 PPT 입력 템플릿이다.

구성은 반드시 아래 흐름을 따른다.

```text
1. 표지
2. 프로필
3. 경험 1 소개
4. 경험 1 결과/KPI
5. 경험 2 소개
6. 경험 2 결과/KPI
...
```

경험이 늘어나면 `경험 소개 → 결과/KPI` 슬라이드를 뒤에 계속 붙인다.

# Slide 1. 표지

## Portfolio

### 레이아웃

```text
[중앙 상단]
Portfolio

[중앙]
[헤드라인]

[중앙 하단]
[포트폴리오 설명 문장]

[하단 연락처 바]
Name | Mobile | E-mail | Web
```

### 입력 내용

```text
헤드라인: [성장하고 확장하는 김미리입니다.]
설명 문장: [포트폴리오 전체를 설명하는 짧은 문장]
Name: [이름]
Mobile: [전화번호]
E-mail: [이메일]
Web: [웹사이트 또는 GitHub]
```

### 작성 기준

- 헤드라인은 지원자 정체성이 드러나는 한 문장으로 작성한다.
- 연락처는 사용자가 제공한 정보만 넣는다.
- 없는 웹사이트나 깃허브 주소는 만들지 않는다.

# Slide 2. 프로필

## Profile

### 레이아웃

```text
[왼쪽 세로 영역]
프로필 사진
안녕하세요
[이름]입니다.
About
이름 / 생년월일 / 이메일 / 전화번호

[오른쪽 상단]
자기소개 요약 문장

[오른쪽 본문 2열]
Education | Experience
Licenses  | Skills
```

### 입력 내용

```text
프로필 사진: [이미지 경로 또는 placeholder]
인사말: [안녕하세요 김미리입니다.]
자기소개 요약: [지원자를 간단히 소개하는 1~2문장]

Education:
- [기간] [학교/전공/상태]

Experience:
- [기간] [경험명/역할]

Licenses:
- [연도] [자격/수상명]

Skills:
- [스킬명] [수준 또는 설명]
```

### 작성 기준

- 프로필 사진 영역은 반드시 유지한다.
- 정보가 없으면 빈 항목으로 두거나 `정보 입력 필요`로 표시한다.
- 장문의 자기소개는 넣지 않는다.

# Slide 3+. 경험 소개

## Experience Intro

### 레이아웃

```text
[상단 중앙]
Project 01

[중앙 제목]
경험 1 : [소제목 역할을 하는 결론 한 줄]

[얇은 구분선]

[3분할 카드]
카드 1: 이미지 / 고객 니즈 / 설명 / 키워드 버튼
카드 2: 이미지 / 문제 OR 기회상황 / 설명 / 키워드 버튼
카드 3: 이미지 / 비교 대상 / 설명 / 키워드 버튼
```

### 입력 내용

```text
Project 번호: [Project 01]
경험 제목: [경험 1 : 자기소개서의 소제목 역할을 하는 결론 한 줄]

고객 니즈:
이미지: [이미지 경로 또는 placeholder]
설명: [1~2줄]
강조 키워드: [짧은 키워드]

문제 OR 기회상황:
이미지: [이미지 경로 또는 placeholder]
설명: [1~2줄]
강조 키워드: [짧은 키워드]

비교 대상:
이미지: [이미지 경로 또는 placeholder]
설명: [1~2줄]
강조 키워드: [짧은 키워드]
```

### 작성 기준

- 3분할 카드 구조를 바꾸지 않는다.
- 카드 제목은 기본적으로 `고객 니즈`, `문제 OR 기회상황`, `비교 대상`을 사용한다.
- 각 설명은 2줄 이내로 작성한다.
- 키워드 버튼은 검정색 계열로 유지한다.

# Slide 4+. 결과/KPI

## Result KPI

### 레이아웃

```text
[상단 중앙]
Project 01

[중앙 제목]
결과물 : [이 경험의 결과를 잘 보여주는 KPI]

[얇은 구분선]

[왼쪽]
차트 / 결과 이미지 / 정성 결과 카드

[가운데]
화살표

[오른쪽 세로 3개 박스]
01. 문제 해결 액션
02. 생산성 증가 액션
03. 의사소통/협상 액션
```

### 입력 내용

```text
Project 번호: [Project 01]
결과 제목: [결과물 : 이 경험의 결과를 잘 보여주는 KPI]

시각 자료 유형: [chart | image | qualitative_card]
시각 자료 내용: [근거 있는 수치, 이미지 설명, 또는 정성 결과]

01. 문제 해결 액션:
- [불릿 1]
- [불릿 2]

02. 생산성 증가 액션:
- [불릿 1]
- [불릿 2]

03. 의사소통/협상 액션:
- [불릿 1]
- [불릿 2]
```

### 작성 기준

- 실제 수치가 있을 때만 차트를 사용한다.
- 수치가 없으면 정성 결과 카드로 대체한다.
- 액션 박스는 3개를 유지한다.
- 각 액션 박스는 1~2개 불릿만 사용한다.

## 최종 점검

- [ ] 템플릿 이름이 `PPT 발표 스펙`인가?
- [ ] 표지와 프로필 슬라이드가 있는가?
- [ ] 경험별로 `경험 소개 → 결과/KPI`가 반복되는가?
- [ ] 경험 소개 슬라이드가 3분할 카드 구조인가?
- [ ] 결과/KPI 슬라이드가 왼쪽 시각 자료, 가운데 화살표, 오른쪽 액션 박스 3개 구조인가?
- [ ] 없는 수치나 성과를 만들지 않았는가?
- [ ] 사진/이미지 영역이 유지되는가?
- [ ] 각 구획의 문장이 1~2문장 이내인가?
- [ ] 문장이 중간에서 어색하게 끊기지 않고 의미 단위로 요약되었는가?
