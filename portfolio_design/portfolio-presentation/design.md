# PPT 발표 스펙 디자인 기준

## 1. 템플릿 목적

PPT 발표 스펙은 지원자의 기본 정보와 프로젝트 경험을 발표용 포트폴리오 흐름으로 보여주는 PPT 형식이다.

이 템플릿은 사용자가 제공한 레퍼런스 구조를 따른다. 표지와 프로필로 지원자를 먼저 소개하고, 이후 경험별로 `경험 소개 → 결과/KPI` 슬라이드를 반복해 발표자가 말로 보충하기 좋은 흐름을 만든다.

## 2. 전체 구조

PPT 발표 스펙은 아래 순서를 기본으로 한다.

| 순서 | 슬라이드 | 역할 |
|---:|---|---|
| 1 | 표지 | 지원자 이름, 포트폴리오 헤드라인, 연락처를 보여준다. |
| 2 | 프로필 | 사진, 기본 정보, 학력, 경험, 자격, 스킬을 요약한다. |
| 3 | 경험 소개 | 경험의 고객 니즈, 문제/기회상황, 비교 대상을 3분할 카드로 보여준다. |
| 4 | 결과/KPI | 해당 경험의 결과, 개선 액션, 핵심 성과를 보여준다. |

경험이 여러 개일 경우 3번과 4번 구조를 경험 수만큼 뒤에 반복한다.

```text
표지
프로필
경험 1 소개
경험 1 결과/KPI
경험 2 소개
경험 2 결과/KPI
...
```

## 3. 디자인 방향

- 출력 형식은 16:9 가로형 PPT를 기본으로 한다.
- 배경은 흰색을 기본으로 하고, 하단에는 아주 연한 하늘색 그라데이션 느낌을 둘 수 있다.
- 전체 분위기는 심플하고 밝게 유지한다.
- 포인트 색상은 하늘색과 검정색을 사용한다.
- 제목은 중앙 정렬을 기본으로 하고, 슬라이드 상단에 작은 프로젝트 라벨을 둔다.
- 각 경험 소개 슬라이드는 3분할 카드 구조를 유지한다.
- 결과/KPI 슬라이드는 왼쪽 시각 자료, 가운데 화살표, 오른쪽 액션 박스 3개 구조를 유지한다.
- 사진은 표지/프로필/경험 소개 카드에 들어갈 수 있다.
- 차트는 결과/KPI 슬라이드에서만 사용한다.
- 실제 수치가 없으면 차트를 임의로 만들지 말고, 정성 결과 카드로 대체한다.
- 하나의 구획에는 1~2문장만 넣는다.
- 문장이 길어 한 줄에 들어가지 않으면 중간에서 끊지 말고 의미 단위로 짧게 다시 요약한다.
- 텍스트 박스 안에서 제목, 설명, 키워드가 서로 겹치지 않도록 최대 줄 수를 지킨다.

## 4. 슬라이드별 구성

### 4.1 표지

#### 포함 내용

- Portfolio 라벨
- 지원자 헤드라인
- 짧은 소개 문장
- 이름, 휴대폰, 이메일, 웹/깃허브 등 연락처

#### 레이아웃 기준

- 화면 중앙에 `Portfolio` 라벨을 작게 배치한다.
- 중앙 큰 제목은 `[성장/확장/문제 해결 등 핵심 키워드]하는 [이름]입니다.` 형식으로 작성한다.
- 제목 아래에는 포트폴리오 전체를 설명하는 짧은 문장을 둔다.
- 하단에는 흰색 연락처 바를 배치하고 `Name | Mobile | E-mail | Web` 순서로 정리한다.
- 불필요한 도형이나 장식은 넣지 않는다.
- 헤드라인은 최대 2줄, 설명 문장은 최대 1줄로 유지한다.

### 4.2 프로필

#### 포함 내용

- 프로필 사진
- 인사말과 이름
- About 정보
- 자기소개 요약 문장
- Education
- Experience
- Licenses
- Skills

#### 레이아웃 기준

- 왼쪽에는 사진과 인사말, 기본 정보를 세로로 배치한다.
- 오른쪽에는 자기소개 문장을 상단에 짧게 배치한다.
- 오른쪽 본문은 2열 구성으로 `Education / Experience / Licenses / Skills`를 배치한다.
- 스킬은 막대 그래프 또는 짧은 텍스트 리스트로 표현한다.
- 모든 항목은 긴 문장보다 날짜와 명사형 중심으로 정리한다.
- 각 목록은 최대 4개 항목만 보여주고, 항목당 한 줄을 넘기지 않는다.

### 4.3 경험 소개

#### 포함 내용

- Project 번호
- 경험 제목
- 고객 니즈
- 문제 OR 기회상황
- 비교 대상
- 각 카드별 이미지
- 각 카드별 짧은 설명
- 강조 포인트 키워드

#### 레이아웃 기준

- 상단 중앙에 `Project 01` 같은 프로젝트 라벨을 둔다.
- 라벨 아래에 `경험 1 : [소제목 역할을 하는 결론 한 줄]` 형태의 제목을 배치한다.
- 제목 아래에는 얇은 구분선을 둔다.
- 본문은 3개의 동일한 폭 카드로 구성한다.
- 각 카드에는 위쪽 이미지, 아래쪽 제목, 1~2줄 설명, 검정색 키워드 버튼을 배치한다.
- 카드 제목은 기본적으로 `고객 니즈`, `문제 OR 기회상황`, `비교 대상`을 사용한다.
- 설명은 발표자가 말로 확장할 수 있게 짧게 작성한다.
- 카드 설명은 최대 2줄, 키워드 버튼은 최대 12자 안팎으로 유지한다.
- 카드 설명이 길면 문장 끝을 말줄임으로 자르지 말고 핵심 명사와 동사를 남겨 다시 쓴다.

### 4.4 결과/KPI

#### 포함 내용

- Project 번호
- 결과 제목
- 결과를 보여주는 시각 자료 또는 정성 결과 카드
- 문제 해결 액션
- 생산성 증가 액션
- 의사소통/협상 액션

#### 레이아웃 기준

- 상단 중앙에 `Project 01` 같은 프로젝트 라벨을 둔다.
- 라벨 아래에 `결과물 : [이 경험의 결과를 잘 보여주는 KPI]` 형식의 제목을 배치한다.
- 제목 아래에는 얇은 구분선을 둔다.
- 왼쪽에는 차트, 지표, 결과 이미지, 또는 정성 결과 카드를 배치한다.
- 가운데에는 오른쪽 흐름을 암시하는 작은 화살표를 둔다.
- 오른쪽에는 세로로 3개의 액션 박스를 배치한다.
- 액션 박스 제목은 기본적으로 `01. 문제 해결 액션`, `02. 생산성 증가 액션`, `03. 의사소통/협상 액션`을 사용한다.
- 각 액션은 1~2개 불릿만 넣는다.
- 각 불릿은 한 줄 중심으로 작성하고, 긴 설명은 발표자가 말로 보충하도록 남긴다.

## 5. AI 작성 원칙

- 템플릿 이름은 항상 `PPT 발표 스펙`으로 사용한다.
- 사용자가 승인한 Wiki, 업로드 자료, 활동 기록에 있는 내용만 사용한다.
- 없는 성과, 수치, 역할, 기술, 회사명은 만들지 않는다.
- 결과/KPI 슬라이드에서 실제 수치가 없으면 차트를 만들지 않는다.
- 텍스트는 발표용으로 짧게 작성한다.
- 한 구획 안에 문장이 많아지면 우선순위가 낮은 문장을 삭제한다.
- 문장을 글자 수 기준으로 억지로 자르지 말고, 문장 전체를 짧은 표현으로 재작성한다.
- 경험이 늘어나면 `경험 소개 → 결과/KPI` 슬라이드 쌍을 뒤에 계속 붙인다.
- 레퍼런스의 표지, 프로필, 3분할 경험 카드, 결과/KPI 구조에서 벗어나지 않는다.

## 6. 피해야 할 것

- 이름을 `상세 기술 포트폴리오`나 `PPT 발표형`으로 다시 쓰는 것
- 개요, 과정, 결과 3장짜리 단순 보고서형으로 바꾸는 것
- 카드 구성을 다른 형태로 재설계하는 것
- 실제 근거가 없는 KPI 차트를 만드는 것
- 한 슬라이드에 많은 문장을 넣어 발표 여백을 없애는 것
- 레퍼런스에 없는 과한 아이콘, 장식, 색상을 추가하는 것
