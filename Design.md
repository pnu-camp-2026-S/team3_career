# Myfitfolio 디자인 문서

## 1. 프로젝트 개요

Myfitfolio는 사용자의 활동 기록과 프로젝트 경험을 바탕으로 포트폴리오를 생성하고 관리할 수 있는 웹 서비스입니다.

사용자는 포트폴리오를 생성한 뒤 관리 화면에서 목록을 확인하고, 좋아요로 중요도를 표시하며, 열기/수정/PPT 저장 기능을 통해 포트폴리오를 활용할 수 있습니다.

## 2. 주요 화면 구성

### 메인

- 서비스 진입 화면입니다.
- 포트폴리오 생성, 포트폴리오 관리, 파일 관리, 활동 추천, 마이페이지로 이동할 수 있습니다.

### 포트폴리오 생성

- 사용자가 포트폴리오 형식, 목적, 반영할 경험을 선택합니다.
- 선택한 정보와 활동 데이터를 바탕으로 포트폴리오 초안을 생성합니다.
- 생성된 초안은 이후 수정하거나 보관함에 저장할 수 있습니다.

### 포트폴리오 관리

- 생성된 포트폴리오 목록을 카드 형태로 보여줍니다.
- 각 카드에는 썸네일, 제목, 생성일, 목적, 한 줄 설명이 표시됩니다.
- 카드 액션은 좋아요, 열기, 수정, PPT 저장으로 구성됩니다.
- 좋아요한 포트폴리오는 목록 상단에 우선 배치됩니다.
- 좋아요하지 않은 포트폴리오는 최신 날짜순으로 정렬됩니다.

### 포트폴리오 웹뷰어

- 포트폴리오 관리 화면에서 `열기`를 누르면 이동하는 화면입니다.
- 기존 텍스트 문서 형태가 아니라 웹 프레젠테이션 뷰어 형태로 구성됩니다.
- 슬라이드는 이미지로 렌더링됩니다.
- 좌우 버튼과 하단 컨트롤바를 통해 슬라이드를 넘길 수 있습니다.
- 우측 상단 닫기 버튼 또는 하단 `목록` 버튼을 통해 포트폴리오 관리 화면으로 돌아갈 수 있습니다.

## 3. 포트폴리오 관리 기능

### 좋아요

- 포트폴리오 카드의 하트 버튼으로 좋아요 상태를 변경합니다.
- 좋아요 상태는 저장됩니다.
- 사용자가 하트를 누르는 즉시 카드 위치가 바뀌면 혼란스러울 수 있으므로, 클릭 순간에는 버튼 상태만 변경됩니다.
- 정렬은 다음 렌더링 또는 재진입 시 반영됩니다.

### 열기

- 선택한 포트폴리오 id를 URL query로 전달합니다.
- 예시: `portfolio_viewer.html?id=portfolio-contest#viewerContent`
- 웹뷰어는 전달받은 id에 해당하는 포트폴리오를 찾아 슬라이드 이미지로 표시합니다.

### 수정

- 선택한 포트폴리오 id를 edit query로 전달합니다.
- 예시: `portfolio_create.html?edit=portfolio-data#pfWorkspaceScreen`
- 포트폴리오 생성 화면에서 기존 내용을 기반으로 수정할 수 있도록 연결합니다.

### PPT 저장

- 포트폴리오 카드에서 `PPT 저장`을 누르면 `.pptx` 확장자 파일 다운로드가 시작됩니다.
- 파일명은 포트폴리오 제목을 기반으로 생성됩니다.
- 현재 단계에서는 기능 확인을 위한 다운로드 흐름을 우선 구현합니다.

## 4. 데이터 구조

포트폴리오 데이터는 브라우저 `localStorage`의 `careerfit_portfolios` 키를 사용합니다.

주요 필드:

- `id`: 포트폴리오 고유 id
- `title`: 포트폴리오 제목
- `createdAt`: 생성일
- `purpose`: 포트폴리오 목적
- `summary`: 카드에 표시할 요약
- `liked`: 좋아요 여부
- `coverLines`: 썸네일 미리보기 문구
- `content`: 웹뷰어와 PPT 저장에 사용할 본문

기능 확인을 위해 기본 샘플 포트폴리오를 제공합니다. 기존 저장 데이터가 적을 경우 누락된 샘플 데이터를 자동으로 합쳐 목록에서 더 많은 항목을 확인할 수 있도록 했습니다.

## 5. UI 디자인 방향

### 포트폴리오 관리

- 전체 화면은 흰 배경과 여백 중심의 관리형 UI로 구성합니다.
- 카드 크기는 너무 작지 않게 조정하여 제목, 설명, 버튼이 안정적으로 보이도록 합니다.
- 액션 버튼은 카드 오른쪽에 배치하여 반복적인 관리 작업이 쉽도록 합니다.
- 좋아요는 텍스트 대신 하트 아이콘으로 표현합니다.

### 웹뷰어

- 레퍼런스와 유사한 웹 프레젠테이션 뷰어 형태를 사용합니다.
- 어두운 배경 위에 슬라이드 이미지를 중앙 배치합니다.
- 좌우 이동 버튼을 슬라이드 양옆에 배치합니다.
- 하단 컨트롤바에는 목록 버튼, 이전/다음 버튼, 현재 슬라이드 번호를 표시합니다.
- 닫기 버튼은 텍스트 기호 대신 CSS 선으로 그려 깜빡임이나 렌더링 아티팩트를 줄입니다.

## 6. 반응형 고려

- 데스크톱에서는 카드 정보와 액션 버튼을 한 줄에 배치합니다.
- 좁은 화면에서는 카드 액션 버튼이 줄바꿈되어 겹치지 않도록 합니다.
- 웹뷰어는 모바일에서 좌우 버튼을 슬라이드 위에 겹쳐 배치하여 공간을 확보합니다.
- 슬라이드 이미지는 16:9 비율을 유지합니다.

## 7. 테스트 및 확인 항목

- 포트폴리오 관리 화면에 카드 목록이 표시되는지 확인합니다.
- 샘플 포트폴리오가 여러 개 표시되는지 확인합니다.
- 좋아요 버튼 상태가 변경되는지 확인합니다.
- 열기 버튼으로 웹뷰어 화면에 이동하는지 확인합니다.
- 웹뷰어에서 슬라이드 이미지가 표시되는지 확인합니다.
- 좌우 버튼과 하단 컨트롤바로 슬라이드 이동이 되는지 확인합니다.
- PPT 저장 클릭 시 `.pptx` 파일 다운로드가 시작되는지 확인합니다.
- `npm test`로 정적 HTML 구조 테스트를 실행합니다.
