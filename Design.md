# Myfitfolio 전체 프로토타입 디자인 문서

## 1. 서비스 개요

Myfitfolio는 대학생과 취업 준비생이 자신의 활동 자료를 정리하고, 부족한 경험을 보완하며, 제출용 포트폴리오를 생성할 수 있도록 돕는 커리어 포트폴리오 관리 서비스입니다.

서비스의 핵심 흐름은 다음과 같습니다.

1. 사용자가 로그인합니다.
2. 메인 화면에서 활동 자료를 업로드하고 폴더별로 관리합니다.
3. 파일 관리 화면에서 자료를 확인하고 AI 요약/분석 흐름을 테스트합니다.
4. 포트폴리오 생성 화면에서 활동 자료를 기반으로 포트폴리오 초안을 만듭니다.
5. 포트폴리오 관리 화면에서 생성된 포트폴리오를 열람, 수정, 저장합니다.
6. 포트폴리오 웹뷰어에서 포트폴리오를 발표 자료처럼 확인합니다.
7. 활동 추천 화면에서 부족한 역량을 보완할 공모전/대외활동/교육/자격증을 확인합니다.
8. 마이페이지에서 개인 정보, 학력, 희망 직무, 관심 분야를 관리합니다.

## 3. 공통 디자인 방향

### 브랜드

- 서비스명: Myfitfolio
- 핵심 인상: 차분한 취업 준비 도구, 학생 친화적 포트폴리오 관리 서비스
- 주요 색상: 보라색 계열 브랜드 컬러
- 보조 색상: 흰색 배경, 연한 회색 라인, 차분한 네이비 텍스트

### 레이아웃

- 대부분의 주요 화면은 상단 내비게이션을 공유합니다.
- 상단 내비게이션에는 메인, 포트폴리오 생성, 포트폴리오 관리, 파일 관리, 활동 추천, 마이페이지가 배치됩니다.
- 화면은 카드와 패널 중심으로 구성하되, 관리형 도구처럼 반복 사용이 편하도록 과도한 장식은 줄입니다.

### 컴포넌트

- 버튼: 주요 액션은 보라색, 보조 액션은 outline 또는 ghost 스타일을 사용합니다.
- 카드: 목록형 정보는 8px 내외의 radius와 얇은 border를 사용합니다.
- 폼: 입력창, select, 검색 버튼은 같은 높이와 간격을 유지합니다.
- 모달/토스트: 파일 관리와 마이페이지에서 상태 안내 및 상세 정보를 보여주는 용도로 사용합니다.

## 4. 화면별 디자인

### 4.1 로그인 / 회원가입

관련 파일:

- `html/index.html`
- `html/login.html`
- `html/signup.html`

목적:

- 사용자가 서비스에 진입하기 전 계정을 생성하거나 로그인하는 화면입니다.

구성:

- Myfitfolio 브랜드 영역
- 이메일/비밀번호 입력
- 소셜 로그인 버튼
- 회원가입 이동

디자인 고려:

- 첫 진입 화면이므로 브랜드명이 명확하게 보여야 합니다.
- 입력 폼은 단순하고 오류 없이 빠르게 사용할 수 있어야 합니다.
- 실제 인증 연동 전 단계에서는 화면 흐름 확인이 우선입니다.

### 4.2 메인 대시보드

관련 파일:

- `html/main.html`

목적:

- 사용자가 업로드한 활동 자료를 전체적으로 확인하고, 활동 유형별 현황을 파악하는 중심 화면입니다.

구성:

- 왼쪽 사이드바
- 활동 폴더 목록
- 자료 업로드 버튼
- 삭제 모드 버튼
- 활동 통계 카드
- AI 전체 분석 패널

사용 흐름:

1. 사용자가 활동 자료를 업로드합니다.
2. 자료가 완료된 활동 폴더 또는 준비된 활동 폴더에 배치됩니다.
3. 메인 영역에서 활동 유형별 통계를 확인합니다.
4. AI 전체 분석 영역에서 전체 자료 분석 흐름을 확인합니다.

디자인 고려:

- 사이드바와 메인 영역의 역할이 명확해야 합니다.
- 폴더와 파일은 반복적으로 추가/삭제될 수 있으므로 목록 UI가 안정적이어야 합니다.
- 통계 카드는 빠르게 스캔할 수 있도록 숫자와 라벨을 분리해 보여줍니다.

### 4.3 파일 관리

관련 파일:

- `html/create.html`

목적:

- 프로젝트 폴더별 자료를 확인하고, AI 요약/분석 흐름을 테스트하는 화면입니다.

구성:

- 파일 관리 헤더
- 전체 자료/분석 완료/선택된 폴더/최근 변경 상태 요약 카드
- 폴더 목록 패널
- 선택 폴더의 자료 목록
- 자료 추가 영역
- AI 정리 상태 패널
- AI 대화창
- 상세 정보 모달
- 완료 토스트

사용 흐름:

1. 사용자가 폴더를 선택합니다.
2. 선택한 폴더의 파일 목록이 표시됩니다.
3. 자료 추가, 미리보기, AI 요약, 삭제 기능을 테스트합니다.
4. 분석하기 버튼을 누르면 AI 정리 상태와 artifact 상태가 변경됩니다.
5. AI 대화창에서 자료 정리 방향을 입력할 수 있습니다.

디자인 고려:

- 실제 저장 없이도 파일 관리 흐름을 이해할 수 있어야 합니다.
- 폴더 목록, 파일 목록, AI 상태가 서로 연결된 것처럼 보여야 합니다.
- 관리 화면이므로 카드형 랜딩 느낌보다 작업 도구 느낌이 우선입니다.

### 4.4 포트폴리오 생성

관련 파일:

- `html/portfolio_create.html`

목적:

- 활동 자료와 프로젝트 경험을 기반으로 제출용 포트폴리오 초안을 생성하는 화면입니다.

구성:

- 포트폴리오 생성 설정
- 포트폴리오 형식 선택
- 사용 목적 선택
- 반영할 경험 선택
- 강조 키워드 선택
- 생성 로딩 화면
- 생성된 포트폴리오 초안
- 다시 설정
- 보관함 저장
- PPT 다운로드
- AI 실시간 수정 대화창

사용 흐름:

1. 사용자가 포트폴리오 형식을 선택합니다.
2. 지원 목적과 반영 경험을 선택합니다.
3. 강조 키워드를 선택합니다.
4. 포트폴리오 생성 버튼을 누릅니다.
5. 로딩 진행률이 표시됩니다.
6. 생성된 초안이 표시됩니다.
7. 사용자는 AI 대화창에 수정 요청을 입력하거나 보관함에 저장합니다.

디자인 고려:

- 설정 단계와 생성 결과 단계가 분명히 구분되어야 합니다.
- AI 실시간 수정은 평소에는 대화하듯 사용하고, 반영 요청 시 초안 수정으로 이어지는 흐름을 상상할 수 있어야 합니다.
- 생성된 결과물은 읽기 쉬운 문단 구조로 보여야 합니다.

### 4.5 포트폴리오 관리

관련 파일:

- `html/portfolio_manage.html`

목적:

- 생성된 포트폴리오를 한 곳에서 확인하고 관리하는 화면입니다.

구성:

- 포트폴리오 관리 제목
- 새 포트폴리오 만들기 버튼
- 포트폴리오 카드 목록
- 카드 썸네일
- 포트폴리오 제목, 생성일, 목적, 한 줄 설명
- 좋아요 버튼
- 열기 버튼
- 수정 버튼
- PPT 저장 버튼

사용 흐름:

1. 사용자가 포트폴리오 관리 화면에 진입합니다.
2. 저장된 포트폴리오 목록이 표시됩니다.
3. 좋아요를 눌러 중요 포트폴리오를 표시합니다.
4. 열기를 눌러 웹뷰어로 확인합니다.
5. 수정을 눌러 포트폴리오 생성/수정 화면으로 이동합니다.
6. PPT 저장을 눌러 `.pptx` 파일 다운로드를 시작합니다.

디자인 고려:

- 여러 포트폴리오를 한 번에 확인할 수 있어야 합니다.
- 카드가 너무 작으면 버튼과 글자가 답답해지므로 적당한 카드 높이와 글자 크기를 유지합니다.
- 좋아요 클릭 직후 카드 위치가 바뀌면 사용자가 혼란스러울 수 있으므로 클릭 순간에는 버튼 상태만 바뀌게 합니다.
- 샘플 포트폴리오를 여러 개 제공해 기능 전반을 확인할 수 있게 합니다.

### 4.6 포트폴리오 웹뷰어

관련 파일:

- `html/portfolio_viewer.html`

목적:

- 포트폴리오를 단순 텍스트가 아니라 발표 자료처럼 확인하는 웹뷰어 화면입니다.

구성:

- 어두운 배경
- 중앙 슬라이드 이미지
- 좌우 이전/다음 버튼
- 하단 컨트롤바
- 현재 슬라이드 번호
- 목록 복귀 버튼
- 우측 상단 닫기 버튼

사용 흐름:

1. 포트폴리오 관리 화면에서 열기 버튼을 누릅니다.
2. 선택한 포트폴리오 id가 query string으로 전달됩니다.
3. 웹뷰어가 해당 포트폴리오 내용을 슬라이드 이미지로 렌더링합니다.
4. 사용자는 좌우 버튼 또는 하단 버튼으로 슬라이드를 넘깁니다.
5. 목록 버튼 또는 닫기 버튼으로 관리 화면에 돌아갑니다.

디자인 고려:

- 레퍼런스처럼 웹 프레젠테이션 뷰어 느낌이 나야 합니다.
- 슬라이드는 HTML 텍스트가 아니라 실제 이미지처럼 보여야 합니다.
- 닫기 버튼은 텍스트 기호 대신 CSS 선으로 그려 깜빡임이나 렌더링 아티팩트를 줄입니다.
- 하단 컨트롤바는 슬라이드 이동 상태를 명확히 보여야 합니다.

### 4.7 활동 추천

관련 파일:

- `html/contest.html`
- `html/contest.css`
- `html/contest.js`

목적:

- 사용자의 포트폴리오에서 부족한 경험을 보완할 활동을 추천하는 화면입니다.

구성:

- AI 맞춤 추천 헤더
- 활동 유형 탭
- 직무 필터
- 난이도 필터
- 키워드 검색
- 추천 활동 목록
- 활동 상세 추천 이유
- 캘린더
- 저장된 일정 목록

사용 흐름:

1. 사용자가 활동 추천 화면에 진입합니다.
2. 전체 또는 공모전/대외활동/자격증/교육 탭을 선택합니다.
3. 직무, 난이도, 키워드로 추천 활동을 좁힙니다.
4. 활동 카드를 클릭해 상세 추천 이유를 확인합니다.
5. 저장한 활동은 캘린더와 일정 목록에서 확인합니다.

디자인 고려:

- 추천 목록과 캘린더가 한 화면에서 같이 보여야 합니다.
- 사용자가 왜 이 활동을 추천받았는지 이해할 수 있어야 합니다.
- 필터는 상단에서 빠르게 조작 가능해야 합니다.

### 4.8 마이페이지

관련 파일:

- `html/mypage.html`

목적:

- 사용자 기본 정보, 학력, 희망 직무, 관심 분야를 관리하는 화면입니다.

구성:

- 기본 정보
- 프로필 사진 변경
- 이름/성별/생년월일
- 연락처 정보
- 학력
- 희망근무조건
- 관심분야
- 섹션 anchor 메뉴
- 검색/선택 모달

사용 흐름:

1. 사용자가 마이페이지에 진입합니다.
2. 기본 정보와 연락처를 확인합니다.
3. 학력 항목을 추가하거나 수정합니다.
4. 희망 직무, 세부 직무, 업종, 지역, 연봉을 선택합니다.
5. 관심 분야, 관심 기업, 관심 산업을 입력합니다.

디자인 고려:

- 입력 항목이 많으므로 섹션별로 명확히 분리합니다.
- 오른쪽 anchor 메뉴로 긴 폼을 빠르게 이동할 수 있어야 합니다.
- 선택형 데이터는 chip과 검색 모달로 관리합니다.

## 5. 데이터와 상태 관리

현재 프로토타입은 대부분 브라우저 내 상태와 샘플 데이터를 사용합니다.

주요 저장 방식:

- `localStorage`
- 화면 내부 JavaScript 배열
- 정적 샘플 데이터

주요 localStorage 키:

- `careerfit_portfolios`: 생성/저장된 포트폴리오 목록

포트폴리오 데이터 구조:

```text
id        포트폴리오 고유 id
title     제목
createdAt 생성일
purpose   목적
summary   카드 요약
liked     좋아요 여부
coverLines 썸네일 문구
content   본문
```

## 6. 프로토타입 범위

현재 프로토타입에서 확인 가능한 것:

- 화면 간 이동
- 자료 업로드 UI
- 폴더/파일 관리 UI
- AI 분석 상태 변화 시뮬레이션
- 포트폴리오 생성 흐름
- 포트폴리오 관리 카드 목록
- 포트폴리오 웹뷰어
- 활동 추천 필터/목록/캘린더
- 마이페이지 입력 UI

현재 프로토타입에서 제한되는 것:

- 실제 서버 DB 저장
- 실제 소셜 로그인
- 실제 AI API 응답 연동
- 실제 완성형 PPTX 생성
- 실제 파일 업로드 영구 저장

## 7. 반응형 고려

- 상단 내비게이션은 작은 화면에서 가로 스크롤 또는 줄바꿈을 고려합니다.
- 카드/패널 레이아웃은 모바일에서 단일 열로 전환합니다.
- 포트폴리오 웹뷰어는 모바일에서 좌우 버튼을 슬라이드 위에 겹쳐 배치합니다.
- 폼 화면은 입력 필드가 세로로 쌓이도록 구성합니다.

## 8. 테스트 및 검증

정적 테스트:

```bash
npm test
```

확인 항목:

- 주요 HTML 파일이 존재하는지 확인
- 서버가 `html/` 폴더를 정적 루트로 제공하는지 확인
- 포트폴리오 생성 로딩 구조 확인
- 포트폴리오 관리 화면 구조 확인
- 포트폴리오 웹뷰어 연결 구조 확인

수동 확인 항목:

- 로그인 후 메인 화면 이동
- 메인에서 파일 업로드/폴더 상태 확인
- 파일 관리에서 폴더 선택, 자료 추가, AI 요약 상태 확인
- 포트폴리오 생성 설정 및 생성 흐름 확인
- 포트폴리오 관리 목록, 좋아요, 열기, 수정, PPT 저장 확인
- 웹뷰어에서 이미지 슬라이드와 좌우 이동 확인
- 활동 추천 필터와 캘린더 확인
- 마이페이지 입력 및 섹션 이동 확인
