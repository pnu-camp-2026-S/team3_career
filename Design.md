# Myfitfolio 전체 프로토타입 Design.md

## 1. 서비스 요약

| 항목 | 내용 |
|---|---|
| 서비스명 | Myfitfolio |
| 대상 사용자 | 대학생, 취업 준비생, 포트폴리오 제출이 필요한 사용자 |
| 핵심 목적 | 활동 자료를 정리하고, 부족한 경험을 보완하며, 제출용 포트폴리오를 생성/관리할 수 있게 돕는 서비스 |
| 주요 기능 | 활동 자료 관리, AI 분석 흐름, 포트폴리오 생성, 포트폴리오 관리, 웹뷰어, 활동 추천, 마이페이지 |
| 프로토타입 성격 | 실제 서비스 전 단계에서 화면 흐름과 주요 기능 동작을 확인하기 위한 HTML/CSS/JavaScript 기반 프로토타입 |

## 2. 전체 사용자 흐름

| 순서 | 사용자 행동 | 연결 화면 | 결과 |
|---:|---|---|---|
| 1 | 로그인 또는 회원가입을 한다 | `index.html`, `login.html`, `signup.html` | 서비스에 진입한다 |
| 2 | 활동 자료를 업로드하고 폴더별로 확인한다 | `main.html` | 활동 자료 현황과 통계를 확인한다 |
| 3 | 폴더와 파일을 관리하고 AI 정리 상태를 확인한다 | `create.html` | 자료 요약, 분석, 대화 흐름을 테스트한다 |
| 4 | 포트폴리오 생성 조건을 선택한다 | `portfolio_create.html` | 목적에 맞는 포트폴리오 초안을 생성한다 |
| 5 | 생성된 포트폴리오를 목록에서 관리한다 | `portfolio_manage.html` | 좋아요, 열기, 수정, PPT 저장을 수행한다 |
| 6 | 포트폴리오를 웹뷰어로 확인한다 | `portfolio_viewer.html` | 이미지 슬라이드 형태로 포트폴리오를 넘겨본다 |
| 7 | 부족한 역량을 보완할 활동을 탐색한다 | `contest.html` | 공모전, 대외활동, 교육, 자격증을 추천받는다 |
| 8 | 개인 정보와 희망 조건을 관리한다 | `mypage.html` | 프로필, 학력, 희망 직무, 관심 분야를 저장한다 |

## 3. 화면 구조

| 화면 | 파일 | 화면 목적 | 핵심 구성 |
|---|---|---|---|
| 로그인 | `html/index.html`, `html/login.html` | 사용자가 서비스에 진입하기 전 로그인한다 | 브랜드, 이메일 입력, 비밀번호 입력, 소셜 로그인, 회원가입 이동 |
| 회원가입 | `html/signup.html` | 신규 사용자가 계정을 생성한다 | 계정 입력 폼, 가입 버튼 |
| 메인 | `html/main.html` | 활동 자료와 전체 현황을 확인한다 | 사이드바, 폴더 목록, 자료 업로드, 활동 통계, AI 전체 분석 |
| 파일 관리 | `html/create.html` | 프로젝트 폴더별 자료를 관리한다 | 폴더 목록, 자료 목록, 자료 추가, AI 정리 상태, AI 대화창 |
| 포트폴리오 생성 | `html/portfolio_create.html` | 활동 자료 기반 포트폴리오 초안을 생성한다 | 생성 설정, 경험 선택, 키워드 선택, 로딩, 초안, AI 수정 |
| 포트폴리오 관리 | `html/portfolio_manage.html` | 생성된 포트폴리오를 목록으로 관리한다 | 카드 목록, 좋아요, 열기, 수정, PPT 저장 |
| 포트폴리오 웹뷰어 | `html/portfolio_viewer.html` | 포트폴리오를 발표 자료처럼 확인한다 | 슬라이드 이미지, 좌우 버튼, 하단 컨트롤바, 목록 이동 |
| 활동 추천 | `html/contest.html` | 부족한 경험을 보완할 활동을 추천한다 | 추천 목록, 필터, 검색, 상세 추천 이유, 캘린더 |
| 마이페이지 | `html/mypage.html` | 사용자 개인 정보와 취업 조건을 관리한다 | 기본 정보, 연락처, 학력, 희망근무조건, 관심분야 |

## 4. 공통 UI 정책

| 구분 | 디자인 방향 |
|---|---|
| 브랜드 컬러 | 보라색 계열을 주요 액션과 강조 요소에 사용 |
| 배경 | 흰색과 연한 회색을 중심으로 관리형 도구 느낌 유지 |
| 카드 | 8px 내외 radius, 얇은 border, 은은한 shadow 사용 |
| 버튼 | 주요 버튼은 보라색, 보조 버튼은 outline 또는 ghost 스타일 사용 |
| 내비게이션 | 상단 nav에서 주요 화면으로 이동 가능하게 구성 |
| 폼 | 입력, select, 검색 버튼의 높이와 간격을 통일 |
| 반응형 | 작은 화면에서 카드와 폼이 세로로 쌓이도록 구성 |

## 6. 주요 데이터 구조

### 6.1 포트폴리오 데이터

| 필드 | 타입 예시 | 설명 |
|---|---|---|
| `id` | string | 포트폴리오 고유 id |
| `title` | string | 포트폴리오 제목 |
| `createdAt` | ISO date string | 생성일 |
| `purpose` | string | 지원 목적 또는 사용 목적 |
| `summary` | string | 관리 카드에 표시할 요약 |
| `liked` | boolean | 좋아요 여부 |
| `coverLines` | string[] | 썸네일 미리보기 문구 |
| `content` | string | 웹뷰어와 PPT 저장에 사용할 본문 |

### 6.2 저장 방식

| 데이터 | 저장 위치 | 설명 |
|---|---|---|
| 포트폴리오 목록 | `localStorage.careerfit_portfolios` | 생성/관리/뷰어에서 공유 |
| 파일 관리 샘플 | 화면 내부 JavaScript 배열 | 프로토타입용 폴더/파일 상태 |
| 활동 추천 데이터 | `contest.js` 내부 데이터 | 추천 목록과 캘린더 표시 |
| 마이페이지 입력값 | 화면 내부 상태 | 프로토타입 단계의 입력 흐름 확인 |

## 7. 프로토타입 범위

| 구분 | 포함 여부 | 설명 |
